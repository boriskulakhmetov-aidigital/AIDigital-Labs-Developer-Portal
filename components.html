Overview

The design system is published as @boriskulakhmetov-aidigital/design-system on GitHub Packages. It provides a unified component library, theme engine, Supabase integration, auth utilities, and access control for all AI Digital Labs applications.

Entry Points

Import PathContents
. (main)All React components, hooks, Supabase integration, types
./style.cssFull CSS bundle (theme variables + component styles)
./cssTheme variables only (no component styles)
./themesTheme definitions, applyTheme, resolveTheme, ThemeConfig types
./utilsrenderMarkdown, download helpers, slugify, SSE parser, theme validation
./loggercreateLogger — structured activity & error logging
./accesscheckAccess, recordUsage, getUserOrgId — tier-based access
./serverServer-only: requireAuth, isAdminUser, API key management
./creativeCreative engine registry for AI image generation
Requires GitHub Packages auth
Add .npmrc with @boriskulakhmetov-aidigital:registry=https://npm.pkg.github.com and a valid NPM_TOKEN.

CSS Architecture

The design system ships two CSS entry points:

ImportContentsWhen to Use
@boriskulakhmetov-aidigital/design-system/style.css Theme variables + all component styles All apps (default)
@boriskulakhmetov-aidigital/design-system/css Theme variables only When you only need CSS variables without component styles

Variable Naming

CSS
/* Background layers */
--bg          /* Page background */
--surface     /* Card/panel background */
--surface2    /* Elevated surface */

/* Borders */
--border      /* Default border */

/* Text */
--text        /* Primary text */
--text-muted  /* Secondary text */

/* Brand */
--accent      /* Primary accent color */

Class Prefix Convention

All design system component classes use the aidl- prefix (e.g., aidl-chat, aidl-sidebar, aidl-report-shell). Micro-report classes use mr- prefix. This prevents collisions with app-specific styles.

Note on legacy variable names
Some older apps use --surface-2 (hyphenated). If you see that, alias it to --surface2 in local CSS.
