Developer Documentation

Build powerful AI-driven marketing analytics into your products. Website audits, neuromarketing scoring, prompt engineering, AI search optimization, and synthetic focus groups — all via API.

Getting Started

  1. Request an API key from your AI Digital Labs account manager
  2. Or generate one in the Admin Console
  3. Your key starts with aidl_ — keep it secret
  4. All API calls use the X-API-Key header

Quick Start (30 seconds)

bash 
# Submit a website audit
curl -X POST https://website-audit.apps.aidigitallabs.com/api/v1/submit \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"website_url": "https://example.com"}'

# Check status (returns job_id)
curl https://website-audit.apps.aidigitallabs.com/api/v1/status/JOB_ID \
  -H "X-API-Key: YOUR_KEY"

# Get result
curl https://website-audit.apps.aidigitallabs.com/api/v1/result/JOB_ID \
  -H "X-API-Key: YOUR_KEY"

Authentication

        
Every API request must include a valid API key in the X-API-Key header.


        

          

            
PropertyDetails

            
              
Formataidl_ followed by 64 hexadecimal characters

              
HeaderX-API-Key: aidl_xxxx...

              
ScopeKeys are scoped to specific apps

              
Rate limitingPer minute and per day (see Rate Limits)

              
ManagementAdmin Console

            
          

        


        

          
Security

          Never expose your API key in client-side code. Always make API calls from your server.
        

      


      
      

        
API Reference — Common Pattern

        
All five tools follow the same asynchronous three-step pattern:


        

          

            
Step 1

            
Submit

            
POST /api/v1/submit

            
Returns job_id

          

          
→

          

            
Step 2

            
Poll

            
GET /api/v1/status/{job_id}

            
Returns status + progress

          

          
→

          

            
Step 3

            
Result

            
GET /api/v1/result/{job_id}

            
Returns full report

          

        


        

          
Tip

          Poll the status endpoint every 5 seconds. Reports are long-running AI analyses that typically take 1–15 minutes depending on the tool.

Response Format

        
All tools return the same response structure when a job completes:


        

          

            json
            
          

          
{
  "job_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "status": "complete",
  "has_visual_report": true,
  "markdown_report": "# AUDIT REPORT\n\n## Executive Summary\n...",
  "visual_report": {
    "schema_version": "2.0",
    "sections": [...]
  },
  "report_url": "https://website-audit.apps.aidigitallabs.com/r/abc123?theme=client"
}

        


        
Response Fields

        

          

            markdown_report
            Raw markdown text. Works everywhere — paste into documents, render in your UI, or feed to another AI.
          

          

            visual_report
            Structured JSON with typed sections, scores, and data. Build your own custom UI on top of this.
          

          

            report_url
            Hosted interactive report URL. Supports white-label theming via the ?theme= parameter.
          

        


        
Status Values

        

          

            
StatusDescription

            
              
queuedJob received, waiting to start

              
processingAI analysis in progress

              
completeReport ready — fetch with /result endpoint

              
errorJob failed (check error field for details)

Code Examples


        

          

            
            
            
          


          

            

              

                python
                
              

              
import requests, time

API_KEY = "aidl_your_key_here"
BASE = "https://website-audit.apps.aidigitallabs.com"

# Submit
r = requests.post(
    f"{BASE}/api/v1/submit",
    headers={"X-API-Key": API_KEY, "Content-Type": "application/json"},
    json={
        "website_url": "https://example.com",
        "instructions": "Focus on mobile conversion funnel"
    }
)
job_id = r.json()["job_id"]
print(f"Job submitted: {job_id}")

# Poll until complete
while True:
    s = requests.get(
        f"{BASE}/api/v1/status/{job_id}",
        headers={"X-API-Key": API_KEY}
    ).json()
    print(f"Status: {s['status']}")
    if s["status"] == "complete":
        break
    if s["status"] == "error":
        raise Exception(f"Job failed: {s.get('error')}")
    time.sleep(5)

# Get result
result = requests.get(
    f"{BASE}/api/v1/result/{job_id}",
    headers={"X-API-Key": API_KEY}
).json()

print(f"\nReport URL: {result['report_url']}")
print(f"Markdown: {len(result['markdown_report'])} chars")
print(f"Visual report available: {result['has_visual_report']}")

            

          


          

            

              

                javascript
                
              

              
const API_KEY = 'aidl_your_key_here';
const BASE = 'https://website-audit.apps.aidigitallabs.com';

async function runAudit(url) {
  // Submit
  const { job_id } = await fetch(`${BASE}/api/v1/submit`, {
    method: 'POST',
    headers: {
      'X-API-Key': API_KEY,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ website_url: url }),
  }).then(r => r.json());

  console.log(`Job submitted: ${job_id}`);

  // Poll
  let status;
  do {
    await new Promise(r => setTimeout(r, 5000));
    status = await fetch(
      `${BASE}/api/v1/status/${job_id}`,
      { headers: { 'X-API-Key': API_KEY } }
    ).then(r => r.json());
    console.log(`Status: ${status.status}`);
  } while (status.status !== 'complete' && status.status !== 'error');

  if (status.status === 'error') {
    throw new Error(`Job failed: ${status.error}`);
  }

  // Result
  return fetch(
    `${BASE}/api/v1/result/${job_id}`,
    { headers: { 'X-API-Key': API_KEY } }
  ).then(r => r.json());
}

runAudit('https://example.com').then(console.log);
